## 📋 STEP-BY-STEP ACTIVATION GUIDE

### STEP 1: Go to GitHub Repository Settings

1. Open your browser
2. Go to: **https://github.com/abremers/biogradix**
3. Click **Settings** (gear icon in top right)

### STEP 2: Navigate to Pages Section

1. In the left sidebar, find **Pages** (scroll down)
2. Click on **Pages**

### STEP 3: Configure GitHub Pages

Under "Build and deployment":

1. **Source** dropdown: Select **Deploy from a branch**
2. **Branch** dropdown: Select **main** (should show "main")
3. **Folder** dropdown: Select **/ (root)**
4. Click **Save**

### STEP 4: Wait for Deployment

1. GitHub will show a blue banner: "GitHub Pages is being built..."
2. Wait 1-2 minutes for it to complete
3. You'll see a green checkmark when done
4. It will show: "Your site is published at https://biogradix.com"

### STEP 5: Verify Your Website

1. Open https://biogradix.com/ in your browser
2. You should see the homepage with Spanish content by default
3. Click **EN** button in top right to see English version
4. Click Footer links: **Términos** and **Privacidad**
5. Everything should work!

## 🔍 TROUBLESHOOTING

### Issue: Still seeing GitHub 404 or repository page

**Solution**: 
1. Make sure you followed all steps above
2. Verify the CNAME file exists (should contain: biogradix.com)
3. Wait 2-3 minutes after saving
4. Hard refresh browser (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac)

### Issue: Domain not working

**Solution**:
1. GoDaddy DNS should already be configured
2. If not, you may need to add GitHub nameservers in GoDaddy:
   - `ns-1234.awsdns-ab.com`
   - `ns-5678.awsdns-cd.com`
   - `ns-9012.awsdns-ef.com`
   - `ns-3456.awsdns-gh.com`
3. Wait 24-48 hours for DNS propagation

### Issue: English version not showing

**Solution**:
1. Click the **EN** button in top right corner
2. Your preference will be saved
3. Refresh page - it should remember English
